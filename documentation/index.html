This page gives just a few guidelines to run the code from the command line. Visit also our github page.

For a full description of the algorithm, please refer to this paper.

Running BBarolo through the GUI allows the user to not learn the annoying list of available parameters: the user has just to fill in the required fields in the GUI and this will create a text file with the correct parameters and launch BBarolo.
However, if you experience any issues with the GUI or you prefer to use the command line tool, in this page you can find a list with the default values and a short description of the main available parameters.


BBarolo can be run by typing in the shell: Bbarolo -p paramfile. The paramfile is a text file with a list of all the needed parameters, in the form:

PARNAME1   VALUE1
PARNAME2   VALUE2
….

An example of parameter file can be downloaded here. Full runnable examples can be found in this page.



General parameters


FITSFILE [none]: The name of the input FITS file. This is a mandatory parameter.

VERBOSE [true]: It enables all the output messages.

BEAMFWHM [none]: The FWHM of the beam in arcsec. This parameter should be not used if the header of the input file is complete. In any case, the BEAMFWHM is ignored if BBarolo finds the keywords BMAJ, BMIN and BPA in the header.

OUTFOLDER [./output]: Set the directory where the output files will be written.


3D Fitting parameters


GALFIT [true]: This flag enables the 3D fitting algorithm.

NRADII [none]: The number of rings to be used and fitted. If not specified, BBarolo will try to estimate it.

RADSEP [none]: The separation between rings in arcsec. If N radii have been requested, the rings will be placed at N*RADSEP + RADSEP/2. If not specified, BBarolo will assume the FWHM of the beam major axis as radius separation.

RADII [none]: This parameter can be used as an alternative to NRADII and RADSEP. The user can provide a text file with a list of rings in arcsec, in the format "file(filename.txt,a,b)", where a is the column number (starting from 1), and b the starting row (all rows if omitted).

XPOS [none]: Global initial value for the x-center of rings. Accepted format are in pixels (starting from 0, unlike GIPSY) or in WCS coordinates in the format +000.0000d (degrees) or +00:00:00.00 (sexagesimal). If not specified, BBarolo will try to estimate it.

YPOS [none]: Like XPOS, but for the y-axis.

VSYS [none]: Global initial value for the systemic velocity in km/s. A text file with a list of values is also accepted (see RADII). If not specified, BBarolo will try to estimate it from the global line profile.

VROT [none]: Global initial value for the rotation velocity in km/s. A text file with a list of values is also accepted (see RADII). If not specified, BBarolo will try to estimate it.

VDISP [8]: Global initial value for the velocity dispersion in km/s. A text file with a list of values is also accepted (see RADII). If not specified, BBarolo will assume 8 km/s.

INC [none]: Global initial value for the inclination in degrees. A text file with a list of values is also accepted (see RADII). If not specified, BBarolo will try to estimate it from the column density map.

PA [none]: Global initial value for the position angle in degrees [0:360], measured from anti-clockwise form the North direction. A text file with a list of values is also accepted (see RADII). If not specified, BBarolo will try to estimate it from the velocity field.

Z0 [none]: Global initial value for scale-height of the disc in arcsec. A text file with a list of values is also accepted (see RADII). If not specified, BBarolo will assume 150 pc after having estimated the distance from the VSYS (or you can set the DISTANCE parameter).

DENS [1]: Gas surface density in units of E20 atoms/cm2. A text file with a list of values is also accepted (see RADII). This parameter is not relevant if a normalization is used.

DELTAINC [5]: This parameter fixes the boundaries of parameter space at [INC-DELTAINC, INC+DELTAINC]. It is not advisable to let the inclination varying over the whole range [0,90].

DELTAPA [15]: This parameter fixes the boundaries of parameter space at [PA-DELTAINC, PA+DELTAPA]. It is not advisable to let the position angle varying over the whole range [0,360].

FREE [VROT VDISP INC PA]: The list of parameters to fit.

FTYPE [2]: Function to be minimized. Accepted values are: 1 (chi-squared), 2 (|mod-obs|, default), 3 (|mod-obs|/(mod+obs)).

WFUNC [2]: Weighting function to be used in the fit. Accepted values are: 0 (uniform weight), 1 (|cos(θ)|) and 2 (cos(θ)^2, default), where θ is the azimuthal angle (=0 for galaxy major axis).

LTYPE [1]: Layer type along z. Accepted values are: 1 (gaussian, default), 2 (sech2), 3 (exponential), 4 (Lorentzian), 5 (box).

SIDE [B]: Side of the galaxy to be fitted. Accepted values are: A (approaching), R (receding), B (both, default)

MASK [SMOOTH]: This parameter tells the code how to build the mask for constraining the regions of true galaxy emission. Accepted values are SMOOTH, SEARCH, THRESHOLD, NONE of a file. If SMOOTH, the input cube is smoothed by a factor FACTOR (default=2) and mask built from the region at S/N>BLANKCUT (default=3). If SEARCH, the source finding will be used to determine the mask. The source finding parameters can be set to change the default values. If THRESHOLD, the cut threshold must be specified through the THRESHOLD parameter in the same flux units of the input datacube. If NONE, all regions with flux>0 will be used. A mask fits file (i.e. filled with 0,1) is also accepted using the syntax file(fitsname.fits).

NORM [LOCAL]: Type of normalization of the model. Accepted values are: LOCAL (pixel by pixel) or AZIM (azimuthal) or NONE.

TWOSTAGE [true]: This flag enables the second fitting stage after parameter regularization. This is relevant just if the user wishes to fit parameters other than VROT and VDISP. The inclination and the position angle are regularized by polynomials of degree POLYN or a Bezier function (default), while the other parameters by constant functions.

POLYN [-1]: Degree of polynomials for the regularization of inclination and position angles. -1 enables the Bezier function.

BWEIGHT [2]: Exponent of weight for blank pixels. See Sec. 2.4 of reference paper for details.

FLAGERRORS [false]: Whether the code has to estimate the errors. This usually heavily slows down the run.

STARTRAD [0]: This parameter allow the user to start the fit from the given ring.

LINEAR [0.85]: This parameter controls the spectral broadening of the instrument. It is in unit of channels and it represents the σ, not the FWHM. The default is for data that has been hanging smoothed, so that FWHM = 2 channels and σ = FWHM/2.355.


Additional parameters for high-z galaxies (BBarolo > 1.2.1)


For high-z galaxies you need to set two additional parameters.

RESTWAVE [none]: The rest wavelength of the line you want to fit. Units must be the same of the spectral axis of the cube. For example, if we want fit H-alpha and CUNIT3 = "angstrom", set 6563.

REDSHIFT [none]: The redshift of the galaxy.

These two parameters are used to calculate the conversion from wavelengths to velocities. The velocity reference is set to 0 at RESTWAVE*(REDSHIFT+1). VSYS has to be set to 0, but can be also used to fine-tune the redshift. Finally, if these two parameters are not set, BBarolo will use the CRPIX3 as velocity reference and the proper VSYS has to be set based on that.
